# Gherkin

# 1. Autenticación

## Feature: Login con magic link

### Scenario: Solicitud de magic link con usuario existente

```
Given un usuario introduce un email válido registrado
When solicita acceso
Then el sistema envía un magic link al email
```

### Scenario: Solicitud de magic link con usuario no existente

```
Given un usuario introduce un email válido no registrado
When solicita acceso
Then el sistema crea una cuenta de usuario
And envía un magic link al email
```

### Scenario: Generación de nuevo magic link invalida anteriores

```
Given un usuario ha solicitado previamente un magic link
When solicita un nuevo magic link
Then el sistema invalida todos los enlaces anteriores
```

---

### Scenario: Uso correcto del magic link

```
Given un usuario tiene un magic link válido
When accede mediante el enlace
Then el sistema autentica al usuario
And inicia una sesión
And invalida el enlace
```

### Scenario: Uso de magic link expirado

```
Given un usuario tiene un magic link expirado
When intenta usar el enlace
Then el sistema rechaza la autenticación
And muestra un mensaje de error
```

### Scenario: Uso de magic link ya utilizado

```
Given un usuario tiene un magic link ya usado
When intenta usar el enlace
Then el sistema rechaza la autenticación
And muestra un mensaje de error
```

---

# 2. Comunidades

## Feature: Creación de comunidad

### Scenario: Crear comunidad válida

```
Given un usuario autenticado
When crea una comunidad con:
  | nombre válido |
  | descripción válida |
  | imagen válida |
Then el sistema crea la comunidad
And la marca como privada
And asigna al creador como admin
```

### Scenario: Crear comunidad sin imagen

```
Given un usuario autenticado
When intenta crear una comunidad sin imagen
Then el sistema rechaza la creación
```

---

## Feature: Acceso a comunidad

### Scenario: Acceso permitido a miembro

```
Given un usuario es miembro de una comunidad
When accede a la comunidad
Then el sistema permite el acceso
```

### Scenario: Acceso denegado a no miembro

```
Given un usuario no es miembro de una comunidad
When intenta acceder a la comunidad
Then el sistema deniega el acceso
```

---

## Feature: Invitaciones

### Scenario: Generar link de invitación

```
Given un usuario es admin de una comunidad
When genera un link de invitación
Then el sistema crea un link único
And el link está asociado a esa comunidad
```

### Scenario: Uso válido de invitación

```
Given un usuario tiene un link de invitación válido
When accede mediante el link
Then el sistema lo añade como miembro de la comunidad
And registra usuario, comunidad y timestamp
And invalida el link
```

### Scenario: Uso de link ya utilizado

```
Given un link de invitación ya ha sido usado
When un usuario intenta usarlo
Then el sistema rechaza el acceso
```

### Scenario: Uso de link por usuario ya miembro

```
Given un usuario ya pertenece a la comunidad
And tiene un link válido
When intenta usar el link
Then el sistema ignora la acción
```

---

# 3. Proyectos

## Feature: Creación de proyectos

### Scenario: Crear proyecto válido

```
Given un usuario es miembro de una comunidad
When crea un proyecto con al menos una imagen
Then el sistema crea el proyecto
And lo asocia a esa comunidad
```

### Scenario: Crear proyecto sin imágenes

```
Given un usuario es miembro de una comunidad
When intenta crear un proyecto sin imágenes
Then el sistema rechaza la operación
```

### Scenario: Crear proyecto con más de 5 imágenes

```
Given un usuario es miembro de una comunidad
When intenta crear un proyecto con más de 5 imágenes
Then el sistema rechaza la operación
```

---

## Feature: Acceso a proyectos

### Scenario: Acceso permitido

```
Given un usuario es miembro de la comunidad del proyecto
When accede al proyecto
Then el sistema permite visualizarlo
```

### Scenario: Acceso denegado

```
Given un usuario no pertenece a la comunidad del proyecto
When intenta acceder al proyecto
Then el sistema deniega el acceso
```

---

# 4. Estados del proyecto

## Feature: Visibilidad por estado

### Scenario: Proyecto en Draft

```
Given un proyecto está en estado Draft
When un usuario distinto al creador intenta acceder
Then el sistema deniega el acceso
```

### Scenario: Proyecto en Live

```
Given un proyecto está en estado Live
When un miembro de la comunidad accede
Then el sistema permite visualizar el proyecto
```

### Scenario: Proyecto en Inactive

```
Given un proyecto está en estado Inactive
When un miembro de la comunidad accede
Then el sistema permite visualizar el proyecto
```

---

## Feature: Feedback por estado

### Scenario: Feedback permitido en Live

```
Given un proyecto está en estado Live
And un usuario pertenece a la comunidad
When envía feedback
Then el sistema registra el feedback
```

### Scenario: Feedback bloqueado en Draft

```
Given un proyecto está en estado Draft
When un usuario intenta enviar feedback
Then el sistema rechaza la acción
```

### Scenario: Feedback bloqueado en Inactive

```
Given un proyecto está en estado Inactive
When un usuario intenta enviar feedback
Then el sistema rechaza la acción
```

---

## Feature: Recomendaciones

### Scenario: Mostrar recomendación

```
Given un proyecto tiene feedback de la comunidad
When el sistema calcula una recomendación
Then muestra una de:
  | Seguir iterando |
  | Impulsar |
  | Pausar |
```

### Scenario: Recomendación no cambia estado

```
Given un proyecto tiene una recomendación
When se muestra al usuario
Then el estado del proyecto no cambia
```

---

# 5. Perfil de usuario

## Feature: Visualización de perfil

### Scenario: Ver perfil dentro de la misma comunidad

```
Given dos usuarios pertenecen a la misma comunidad
When uno accede al perfil del otro
Then el sistema muestra el perfil
```

### Scenario: Ver perfil sin comunidad compartida

```
Given dos usuarios no comparten comunidad
When uno intenta ver el perfil del otro
Then el sistema deniega el acceso
```

---

## Feature: Edición de perfil

### Scenario: Editar perfil

```
Given un usuario autenticado
When actualiza sus intereses o bio
Then el sistema guarda la información
```

---

## Feature: Información del perfil

### Scenario: Mostrar información del perfil

```
Given un perfil de usuario
When es visualizado
Then el sistema muestra:
  | intereses |
  | bio |
  | número de feedbacks |
  | número de proyectos |
```

---

# 6. Feedback

## Feature: Envío de feedback

### Scenario: Enviar feedback válido

```
Given un usuario pertenece a la comunidad
And el proyecto está en estado Live
When envía feedback
Then el sistema lo asocia al usuario
And lo asocia al proyecto
```

# 7. Seguridad y acceso

## Feature: Protección de acceso

### Scenario: Acceso por URL directa no autorizado

```
Given un usuario no autorizado
When intenta acceder mediante URL directa a una comunidad o proyecto
Then el sistema deniega el acceso
```

---

## Feature: Privacidad por comunidad

### Scenario: Restricción de visibilidad

```
Given un recurso pertenece a una comunidad
When un usuario fuera de la comunidad intenta acceder
Then el sistema deniega el acceso
```
