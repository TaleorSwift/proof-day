# HUs

# 1. Autenticación

## HU-AUTH-001 — Login con magic link

**Como** usuario

**Quiero** introducir mi email para recibir un enlace de acceso

**Para** poder iniciar sesión sin contraseña

### Criterios de aceptación (derivados de EARS-AUTH-001, 002, 004)

- El usuario puede introducir un email válido
- El sistema envía un magic link al email
- Si el usuario no existe, se crea automáticamente
- El enlace tiene una validez máxima de 15 minutos
- El enlace es de un solo uso

---

## HU-AUTH-002 — Uso del magic link

**Como** usuario

**Quiero** acceder mediante el enlace recibido

**Para** iniciar sesión en la plataforma

### Criterios de aceptación (EARS-AUTH-006, 007, 008, 009, 010)

- El usuario puede autenticarse usando un link válido
- El link se invalida tras su uso
- Si el link está expirado, el acceso es rechazado
- Si el link ya fue usado, el acceso es rechazado
- El sistema muestra un mensaje de error claro en caso de fallo

---

## HU-AUTH-003 — Gestión de múltiples links

**Como** usuario

**Quiero** que solo el último enlace sea válido

**Para** evitar usar enlaces antiguos

### Criterios de aceptación (EARS-AUTH-003)

- Al generar un nuevo magic link, los anteriores quedan invalidados

---

# 2. Comunidades

## HU-COM-001 — Crear comunidad

**Como** usuario autenticado

**Quiero** crear una comunidad privada

**Para** colaborar con otros usuarios

### Criterios de aceptación (EARS-COM-001 a 006)

- Se requiere:
    - nombre (1–100 caracteres)
    - descripción (1–300 caracteres)
    - imagen válida
- Si la imagen no es válida o falta, se rechaza la creación
- La comunidad es privada
- El creador se asigna como admin

---

## HU-COM-002 — Acceso restringido a comunidad

**Como** usuario

**Quiero** acceder solo a comunidades de las que soy miembro

**Para** proteger la privacidad

### Criterios de aceptación (EARS-COM-007, 008)

- Usuarios no miembros no pueden acceder
- Usuarios miembros sí pueden acceder

---

## HU-COM-003 — Generar invitación

**Como** admin de comunidad

**Quiero** generar enlaces de invitación

**Para** permitir que otros usuarios se unan

### Criterios de aceptación (EARS-COM-009, 010, 011)

- Se pueden generar múltiples enlaces
- Cada enlace pertenece a una comunidad
- Los enlaces son no predecibles

---

## HU-COM-004 — Unirse a comunidad

**Como** usuario

**Quiero** usar un enlace de invitación

**Para** unirme a una comunidad

### Criterios de aceptación (EARS-COM-012, 013, 016, 017, 018)

- El usuario es añadido como miembro si el link es válido
- El link se consume tras uso
- Si el link ya fue usado, se rechaza
- Si el usuario ya es miembro, se ignora la acción
- Se registra:
    - usuario
    - comunidad
    - timestamp

---

## HU-COM-005 — Validación de invitación

**Como** sistema

**Quiero** validar enlaces de invitación

**Para** evitar accesos inválidos

### Criterios de aceptación (EARS-COM-014, 015)

- Links expirados son rechazados
- Links revocados son rechazados

---

# 3. Proyectos

## HU-PROJ-001 — Crear proyecto

**Como** miembro de una comunidad

**Quiero** crear un proyecto

**Para** compartirlo con la comunidad

### Criterios de aceptación (EARS-PROJ-001, 005, 006)

- El proyecto pertenece a una única comunidad
- El usuario debe ser miembro
- Debe incluir al menos una imagen

---

## HU-PROJ-002 — Validación de imágenes

**Como** sistema

**Quiero** validar la galería del proyecto

**Para** asegurar consistencia

### Criterios de aceptación (EARS-PROJ-007 a 010)

- Máximo 5 imágenes
- Mínimo 1 imagen
- Si no se cumple, se rechaza la operación
- Aplica en creación y edición

---

## HU-PROJ-003 — Acceso a proyectos

**Como** usuario

**Quiero** ver proyectos de mi comunidad

**Para** interactuar con ellos

### Criterios de aceptación (EARS-PROJ-002, 003, INT-001)

- Solo miembros pueden acceder
- No miembros son bloqueados

---

## HU-PROJ-004 — Interacción con proyectos

**Como** usuario

**Quiero** interactuar con proyectos

**Para** aportar feedback

### Criterios de aceptación (EARS-PROJ-004, STATE-004, INT-004)

- Se puede interactuar solo si:
    - pertenece a la comunidad
    - el proyecto está en estado Live

---

# 4. Estados del proyecto

## HU-STATE-001 — Visibilidad por estado

**Como** usuario

**Quiero** ver proyectos según su estado

**Para** entender su disponibilidad

### Criterios de aceptación (EARS-STATE-001, 002, 003)

- Draft → solo visible para el creador
- Live → visible para la comunidad
- Inactive → visible para la comunidad

---

## HU-STATE-002 — Restricción de feedback

**Como** sistema

**Quiero** controlar cuándo se puede dar feedback

**Para** mantener coherencia del flujo

### Criterios de aceptación (EARS-STATE-004, 005, 006)

- Solo se puede dar feedback en estado Live
- En Draft e Inactive está bloqueado

---

## HU-STATE-003 — Recomendaciones

**Como** usuario

**Quiero** ver recomendaciones basadas en feedback

**Para** decidir qué hacer con mi proyecto

### Criterios de aceptación (EARS-STATE-007, 008)

- El sistema muestra una de:
    - Seguir iterando
    - Impulsar
    - Pausar
- No cambia el estado del proyecto

---
